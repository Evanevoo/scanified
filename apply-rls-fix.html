üö® Problem Diagnosis

Issue: New users cannot read join codes due to Row Level Security (RLS) policies.
Symptom: "This join code does not exist" even for valid codes.
Cause: RLS prevents users without an organization from reading the organization_join_codes table.
Solution: Update RLS policies to allow authenticated users to read join codes for validation.

Connect with service role key first

üìã Manual SQL Commands

If the automatic fix doesn't work, run these commands manually in Supabase Dashboard > SQL Editor:

-- Drop existing restrictive policy
DROP POLICY IF EXISTS "Users can view organization join codes" ON organization_join_codes;

-- Create new policy that allows any authenticated user to read join codes for validation
CREATE POLICY "Authenticated users can read join codes for validation" 
ON organization_join_codes 
FOR SELECT 
USING (auth.role() = 'authenticated');

-- Drop old management policy
DROP POLICY IF EXISTS "Organization members can manage join codes" ON organization_join_codes;

-- Create specific policies for management operations
CREATE POLICY "Organization members can create join codes" 
ON organization_join_codes 
FOR INSERT 
WITH CHECK (
  EXISTS (
    SELECT 1 FROM profiles 
    WHERE profiles.id = auth.uid() 
    AND profiles.organization_id = organization_join_codes.organization_id
    AND profiles.role IN ('admin', 'manager', 'owner')
  )
);

CREATE POLICY "Organization members can update join codes" 
ON organization_join_codes 
FOR UPDATE 
USING (
  EXISTS (
    SELECT 1 FROM profiles 
    WHERE profiles.id = auth.uid() 
    AND profiles.organization_id = organization_join_codes.organization_id
    AND profiles.role IN ('admin', 'manager', 'owner')
  )
);

CREATE POLICY "Organization members can delete join codes" 
ON organization_join_codes 
FOR DELETE 
USING (
  EXISTS (
    SELECT 1 FROM profiles 
    WHERE profiles.id = auth.uid() 
    AND profiles.organization_id = organization_join_codes.organization_id
    AND profiles.role IN ('admin', 'manager', 'owner')
  )
);
